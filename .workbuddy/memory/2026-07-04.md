### 功能详情
1. **当前版本检测**：从服务器目录中 bedrock-server-*.zip 文件名解析版本号
2. **最新版本检查**：抓取 minecraft.net 下载页面，解析稳定版和预览版最新版本
3. **下载管理**：后台线程下载 zip，带进度条、取消功能、文件大小显示
4. **升级流程**：
   - 停止服务器（提示确认）
   - 可选备份：worlds/资源包/行为包/配置文件等关键数据
   - 解压新版本覆盖核心文件（跳过用户数据目录）
   - 恢复备份的用户数据
5. **集成**：添加到主窗口 `refresh_all_tabs` 自动刷新

## 修复：版本检查失败 + 添加滚动区域
- **问题**：minecraft.net 页面完全由 JS 渲染，urllib 抓取不到下载链接
- **解决**：改用 HEAD 请求逐版探测方案（从当前版本向上逐个 patch+build 探测）
- CPU 探测逻辑：当前 minor 不变，逐个 patch 递增，每个 patch 探测 build 0~6，连续 2 个空 patch 停止
- 测试结果：从 1.26.31.1 探测到 1.26.32.2（~25s），从最新版探测显示无更新（~13s）
- **UI 修复**：UpgradeTab 整体包裹在 `QScrollArea` 中，小屏幕也可滚动查看

## 优化：并发探测 + 缓存 + 手动输入 + 进度条
- **并发 HEAD 探测**：使用 `ThreadPoolExecutor` (max_workers=8) 替代顺序探测，性能提升 ~4x
  - 8 个 URL：1.6s（并发） vs ~6s（顺序）
- **结果缓存**：最新版本信息存入 `bds_manager_config.json` 的 `version_cache` 字段
  - 缓存有效期 1 小时，过期后重新探测
  - `load_config` / `save_config` 均已支持 `version_cache`
- **手动版本输入**：新增「✏️ 手动指定版本」区块
  - 输入版本号 → 验证格式 → HEAD 确认存在 → 直接下载
  - 支持稳定版/预览版切换
- **实时进度**：探测时显示进度条 + 当前探测的版本号 + 取消检查按钮
  - `CheckWorker` 发射 `progress_signal(str, int)`，UI 实时更新
